# Texture Design Guide

## Approach

Block models reference vanilla textures directly via the `minecraft:` namespace wherever
possible. The only custom texture per wood type is the **book-spine face** of the regular
bookshelf ‚Äî 12 PNG files total.

| Face | Source |
|------|--------|
| Regular bookshelf sides | `project-bookcase:block/<wood>_bookshelf` (custom PNG) |
| Regular bookshelf top/bottom | `minecraft:block/<wood>_planks` (vanilla, referenced in model JSON) |
| Chiseled bookshelf sides | `minecraft:block/<wood>_planks` (vanilla, referenced in model JSON) |
| Chiseled bookshelf top/bottom | `minecraft:block/<wood>_planks` (vanilla, referenced in model JSON) |
| Chiseled bookshelf front (empty) | `minecraft:block/chiseled_bookshelf_empty` (vanilla, referenced in model JSON) |
| Chiseled bookshelf front (occupied) | `minecraft:block/chiseled_bookshelf_occupied` (vanilla, referenced in model JSON) |

The vanilla references are wired up in the block model JSON files generated by data gen ‚Äî
no extra image assets are needed for those faces.

---

## Files to Create

One PNG per wood type, 12 total:

```
<wood>_bookshelf.png
```

Wood types: `oak`, `spruce`, `birch`, `jungle`, `acacia`, `dark_oak`, `mangrove`, `cherry`, `pale_oak`, `crimson`, `warped`, `bamboo`

---

## Specs

- **Size:** 16√ó16 px
- **Format:** PNG, no transparency (fully opaque)
- **Color depth:** 8-bit or 32-bit RGBA ‚Äî Minecraft ignores the alpha channel on solid block faces

---

## How to Create Each Texture

Base each file on vanilla's `bookshelf.png`. The texture is 16√ó16 px with this layout:

```
Row  0  ‚ñë top plank strip    ‚Üê recolor per wood type
Row  1  ‚ñë top plank strip    ‚Üê recolor per wood type
Row  2  ‚ñà book spines
...       (12 rows of book spines ‚Äî identical across all wood types)
Row 13  ‚ñà book spines
Row 14  ‚ñë bottom plank strip ‚Üê recolor per wood type
Row 15  ‚ñë bottom plank strip ‚Üê recolor per wood type
```

Only rows 0‚Äì1 and 14‚Äì15 change between wood types. The book spines (rows 2‚Äì13) are
always kept exactly as they appear in vanilla's `bookshelf.png`.

Each `.txt` placeholder file contains the exact plank palette (hex colors extracted
from `<wood>_planks.png` in 26.1.2.jar) and a visual description for that specific
wood type.

Extract vanilla source textures with:

```bash
cd ~/Library/Application\ Support/minecraft/versions/26.1.2
unzip 26.1.2.jar "assets/minecraft/textures/block/bookshelf.png" \
                  "assets/minecraft/textures/block/*_planks.png" \
                  -d /tmp/mc-textures
open /tmp/mc-textures/assets/minecraft/textures/block/
```

## When Done

Delete the corresponding `.txt` placeholder and commit the `.png` in its place.
